# ONP Frontend Application - Presentation

## Slide 1: Title
**ONP Frontend Application**
Orion Notification Platform - Onboard UI

---

## Slide 2: Overview
**What is ONP Frontend?**
- Web-based interface for Orion Notification Platform
- Simplifies event onboarding and management
- Supports 18+ environments (DEV, QA, INT, PROD, etc.)
- Modern, user-friendly interface

---

## Slide 3: Main Features - 4 Core Modules

**1. New Event Onboarding**
- Configure and onboard new events

**2. MongoDB & Redis Operations**
- View, Update, Insert events

**3. Kafka Management**
- View topics, Create topics

**4. User Guide**
- Built-in documentation and help

---

## Slide 4: Feature 1 - New Event Onboarding
**Complete Event Setup in One Place**

**6 Request Criteria:**
- MongoDB and Redis (Schema storage)
- Kafka Topic (Message streaming)
- Deployment Manifest (Infrastructure)
- Orion Properties (Configuration)
- Fallback DB (Error handling)
- Concourse Vault (CI/CD secrets)

**Key Capabilities:**
- Select multiple criteria at once
- Dynamic form fields based on selection
- JSON schema editors with syntax highlighting
- Real-time validation
- Task execution with status tracking
- Export results to Excel

---

## Slide 5: Feature 2 - MongoDB & Redis Operations
**Three Operations in One Module**

**1. View Events**
- Browse all events or search specific ones
- Complete event details (schemas, downstream, auth)
- Redis cache information
- Export to Excel

**2. Update Events**
- Modify existing event configurations
- Update schemas and downstream settings
- Auto-refresh Redis cache

**3. Insert Events**
- Add new events to MongoDB
- Complete configuration setup
- Automatic cache refresh

---

## Slide 6: Feature 3 - Kafka Management
**Two Operations for Kafka Topics**

**1. View Kafka Details**
- View all topics or specific topics
- Health status (Healthy/Unhealthy/Not Found)
- Partition and replication information
- Consumer groups details
- Topic configuration settings
- Export to Excel

**2. Create Kafka Topics**
- Configure partitions and replication factor
- Environment-specific creation
- Validation before creation
- Status tracking
